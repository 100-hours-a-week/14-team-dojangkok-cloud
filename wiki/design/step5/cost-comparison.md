## 4. 비용 비교: V2 두 환경 vs K8S 한 클러스터

### 인프라 비교 (GPU 제외)

> K8S는 기본 구성에서 FE/BE/AI Server **복제본 2**(서비스 이중화)를 포함합니다.
> 동등 조건 비교를 위해 V2도 **ASG=2(이중화 적용)** 기준으로 산정합니다.

| | 현행 V2 (ASG=2) | K8S |
|--|---------|-----|
| 구성 | 환경당 AWS 9대 + GCP VM **× 2** | 클러스터 1개 (6대), Namespace 분리 |
| 환경 1개 | ~$334 | - |
| **두 환경 합계** | **~$668** | **~$456** |
| **차이** | | **-$212/월 (-32%)** |

### 스케일링 비용 비교

| 시나리오 | 현행 V2 (× 2env) | K8S (1 클러스터) |
|----------|-----------------|-----------------|
| **ASG=1 (이중화 없음)** | ~$596 | - |
| **ASG=2 (이중화, 위 기준)** | ~$668 | ~$456 |
| **FE/BE 각 4 복제본** | ~$812 | ~$456 (Worker 여유분 활용) |
| **Worker 1대 추가** | 해당 없음 | ~$554 (+t4g.xlarge 1대) |

> **핵심 차이**: V2는 스케일링 시 인스턴스 단위로 비용이 선형 증가하지만, K8S는 워커 노드 여유 자원 내에서 Pod를 추가할 수 있어 **소규모 스케일링 시 추가 비용 $0**

## 5. 비용 변동 요인 분석

**증가 요인**:
- Worker 노드 스펙 상향: t4g.small(2GB) → t4g.xlarge(16GB) × 3대
- MongoDB EC2 신규 추가 (채팅 데이터)
- PVC 추가: Redis, RabbitMQ, ChromaDB, 모니터링 스택

**감소 요인**:
- **GCP 인프라 전체 폐기**: AI Server, ChromaDB, 스토리지, 네트워크 = **-$254** (× 2env)
- **NLB 제거**: 크로스 클라우드 통신 불필요 → **-$37** (× 2env)
- **EIP 제거**: NAT Instance 불필요 → **-$15** (× 2env)
- **단일 클러스터**: Dev+Prod를 1세트 인프라로 운영 (V2는 환경별 개별 인프라)

### 비용 외 고려사항

| 항목 | 현행 | K8S 전환 후 |
|------|------|-----------|
| 서비스 이중화 | ASG min=max=1 (실질 이중화 없음) | Pod 복제본 2 (노드 분산) |
| 장애 복구 | VM 재프로비저닝 수 분 | Pod 재스케줄링 수 초 |
| 스케일링 비용 | 인스턴스 단위 선형 증가 | 노드 여유분 내 추가 비용 없음 |
| 환경 관리 | Dev/Prod 인프라 완전 이원화 | Namespace 분리, 인프라 1세트 |
| Terraform 코드베이스 | AWS 12 + GCP 10 모듈 × 2env | AWS 노드 프로비저닝만 |
| 배포 파이프라인 | CodeDeploy + MIG 이원화 | ArgoCD 단일 |
| 모니터링 에이전트 | VM별 수동 설정 | DaemonSet 자동 배포 |
| 컨테이너 레지스트리 | ECR + GAR | ECR 단일 |
