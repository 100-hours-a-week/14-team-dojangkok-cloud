## 4. 비교 요약

### 동등 조건 비교

K8S는 Pod 복제본(FE×2, BE×2)으로 서비스 이중화가 기본 포함.
V2에서 동등 수준은 ASG=2.

| | V2 Prod (ASG=2) | K8S Prod (A) | K8S Prod (B) |
|--|----------------|-------------|-------------|
| 컴퓨트 | $209 (AWS) + $111 (GCP) = $320 | $230 | $230 |
| 스토리지 | $24 | ~$35 | ~$50 |
| 네트워크 | ~$65 | ~$31 | ~$31 |
| **합계** | **~$334** | **~$296** | **~$311** |
| **차이** | — | **-$38 (-11%)** | **-$23 (-7%)** |

### V2 ASG=1 (현재 상태) 비교

| | V2 Prod (ASG=1) | K8S Prod (A) | K8S Prod (B) |
|--|----------------|-------------|-------------|
| **합계** | **~$298** | **~$296** | **~$311** |
| **차이** | — | **-$2 (동등)** | **+$13 (+4%)** |

> V2 ASG=1은 이중화 없는 상태. K8S는 이중화 포함이므로 **같은 비용에 이중화를 얻는 셈**.

### 비용 변동 요인

| 요인 | 변동 | 금액 |
|------|------|------|
| GCP 인프라 전액 제거 | 감소 | -$127 |
| NLB 제거 | 감소 | -$18 |
| NAT 1대 축소 + EIP 1개 축소 | 감소 | -$7 |
| **절감 소계** | | **-$152** |

| 요인 | 변동 | 금액 |
|------|------|------|
| Worker 4대 (t4g.large) | 증가 | +$196 |
| CP 노드 추가 (t4g.medium) | 증가 | +$30 |
| V2 FE/BE/DB 인스턴스 대체 | 감소 | -$98 (V2 ASG=1 컴퓨트) |
| PVC 추가 (DB HA 시) | 증가 | +$15 |
| **순 변동** | | **-$2 ~ +$13** |

## 5. 가치 비교

K8S가 V2와 **비슷한 비용**이면서 추가 가치 포함:

| 가치 | V2 Prod | K8S Prod |
|------|---------|---------|
| 서비스 이중화 | ASG=1 (이중화 없음) | Pod 복제본 2 (기본) |
| DB HA | 단일 인스턴스 | Primary-Replica (시나리오 B) |
| 장애 복구 | VM 재프로비저닝 (수 분) | Pod 재스케줄링 (수 초) |
| 단일 클라우드 | AWS + GCP 이원화 | AWS 통합 |
| 인스턴스 수 | 9대 (AWS 7 + GCP 2) | 6대 |
| 관리 도구 | AWS/GCP 콘솔 + Terraform | kubectl 단일 |
